객체 지향 언어 객체간의 상호작용시 사용하는 언어

객체 :  클래스에 정의된 내용대로 new 연산자를 통해 메모리 영역에 생성된 것

클래스 : 객체의 특성을 정의한 것

객체지향언어의 4가지 특징

-추상화 : 유연성을 확보하기 위해 구체적인 것은 제거

-캡슐화 : 데이터와 기능을 하나로 묶어 클래스 외부에서 데이터의 직접적인 접근을 막음

-상속 : class 다음에 'extends 클래스명' 추가
	모든 클래스는 Object 자식들
	super() : 부모 객체의 생성자를 호출하는 메서드
	오버라이딩 : 상속 받은 부모 메서드를 재정의 하는것
		    하위 클래스에서 메서드 정의/매개변수 동일/ 리턴타입 동일
	오버로딩 :     같은 클래스에서 메서드 정의/ 매개변수 다름/리턴타입 노상관

-다형성 : 다양한 형태를 가짐
	부모 클래스 타입 참조변수 하나로 상속 관계에 있는 여러 타입의 자식 객체를 참조가능
	업 캐스팅 : 상속관계일때 부모타입의 참조형 변수가 모든 자식 타입 객체의 주소를 참조 가능

	다운 캐스팅 : 부모 클래스 타입의 참조형 변수를 자식 클래스 타입으로 형변환
	instanceof 연산자 : 현재 참조형 변수가 어떤 클래스 형의 객체 주소를 참조하고 있는지 확인할 때 사용(true/false)
			


메소드 예약어

- static : static 영역에 할당

final : 중단 상속 불가 클래스
